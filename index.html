        Resumo: Instalação e Atualização do Node.js e Angular CLI

        Neste vídeo, mostramos como instalar ou atualizar o Node.js e o Angular CLI para suas versões mais recentes, garantindo compatibilidade e melhor desempenho.

        Etapas para Instalar/Atualizar:

        1. Desinstalar Versões Antigas (se aplicável):

        • Node.js:

          Vá ao Painel de Controle do Windows, localize o Node.js e clique em Desinstalar.

          Verifique no terminal executando:

          node --version

          Se der erro, o Node foi desinstalado com sucesso.

        • Angular CLI:

          No terminal, remova o pacote globalmente:

          npm uninstall -g @angular/cli

          Verifique com:

          ng version

          Se der erro, o Angular CLI foi removido.

        2. Instalar as Versões Mais Recentes:

        • Node.js:

          Acesse Node.js Downloads e baixe a versão mais recente (atualmente 20.x).

          Siga as etapas do instalador e conclua a instalação. Verifique no terminal:

          node --version
        • Angular CLI:

          Instale a última versão globalmente com o comando:

          npm install -g @angular/cli@latest

          Ou especifique uma versão específica:

          npm install -g @angular/cli@17.0.10

          Verifique a versão instalada:

          ng version

        3. Compatibilidade Angular e Node.js:

        • Angular CLI 17: Compatível com Node.js versões 18 e 20.
        • Regra Geral: Verifique sempre a tabela de compatibilidade antes de instalar versões específicas.

        Dicas Importantes:

        • Sempre desinstale versões antigas antes de atualizar.
        • Mantenha o Node.js e o Angular CLI nas versões mais recentes para evitar problemas.
        • Utilize o comando -version para validar as instalações.
        • O -g significa “globalmente” na hora da instalação

      • Aula 009 - Criação de projetos na Versão 17 (Importante)

        Resumo: Como Criar Projetos no Angular com Compatibilidade Entre Versões

        Este vídeo explica como criar projetos Angular para acompanhar o curso e evitar confusões relacionadas às diferenças entre as versões 16 e 17 do Angular. A abordagem garante compatibilidade e facilita o aprendizado ao seguir os exemplos do curso.

        Contexto

        • O curso foi criado inicialmente utilizando a versão 16 do Angular.
        • A partir da seção Reactive Forms, o curso passa a utilizar a versão 17 do Angular.
        • A versão 17 introduziu o suporte a componentes standalone como padrão ao criar novos projetos. No entanto, os exemplos do curso utilizam a estrutura com AppModule, comum em projetos do mercado.

        Problema

        • Projetos gerados na versão 17 possuem uma estrutura diferente, utilizando o formato standalone.
        • Isso pode causar confusão para iniciantes, já que a estrutura padrão do mercado e dos exemplos do curso utiliza o AppModule.

        Solução: Como Criar Projetos Angular

        1. Criar Projetos Compatíveis com a Versão 16

        • Para gerar um projeto Angular na versão 17, mas com a estrutura similar à versão 16 (com AppModule), utilize a flag:
          ng new nome-do-projeto --no-standalone

        2. Passo a Passo para Criar o Projeto

        1. Abra o terminal (pode ser o do Git ou outro terminal).
        1. Execute o comando para criar o projeto com a flag mencionada:
          ng new nome-do-projeto --no-standalone
        1. Responda às perguntas:
          • Folha de Estilo: Escolha SCSS (segunda opção) pressionando Enter.
          • Server-Side Rendering (SSR): Responda N para "não".
        1. O Angular CLI instalará os pacotes e criará o projeto.

        3. Estrutura do Projeto Criado

        • Com a flag -no-standalone, o projeto incluirá a estrutura com AppModule, garantindo compatibilidade com os exemplos do curso.

        Execução de Projetos Criados

        • Mesmo que você tenha a versão 17 do Angular instalada, os projetos criados com estrutura da versão 16 funcionarão normalmente.
        • Após criar o projeto, entre no diretório, instale as dependências e inicie o servidor:
          cd nome-do-projeto
npm install
ng serve

        Por Que Dominar a Estrutura com AppModule?

        • Relevância no Mercado: A maioria dos projetos Angular no mercado ainda utiliza a estrutura com AppModule, especialmente para projetos maiores ou mais antigos.
        • Praticidade no Curso: Todos os exemplos até a seção Template Driven Forms utilizam essa estrutura.

        Dica Importante

        • Sempre utilize a flag -no-standalone ao criar projetos na versão 17 para manter a compatibilidade e evitar confusão.
        • Caso esteja utilizando a versão 16 do Angular, não há necessidade de usar a flag, pois essa versão já utiliza o AppModule por padrão.

        Conclusão
        Com este método, você poderá criar projetos compatíveis com os exemplos do curso e aprender sem dificuldades. Caso tenha dúvidas, consulte os comentários ou deixe sua pergunta para esclarecimentos.

      • Aula 011 - Instalação de uma Versão Específica do Node.js e Angular CLI

        Resumo: Como Instalar Versões Específicas do Node.js e Angular CLI

        Este vídeo apresenta como instalar uma versão específica do Node.js ou Angular CLI e a importância de garantir a compatibilidade entre essas ferramentas.

        Instalar uma Versão Específica do Node.js

        1. Acesse o site oficial do Node.js

          Node.js

          • Clique na aba de "Downloads".
          • Desça até a seção de "Previous Releases".
        1. Escolha a versão desejada
          • Localize a versão específica que você quer instalar na tabela de versões.
          • Clique na opção de "Releases" da versão selecionada.
        1. Faça o download do instalador
          • Para Windows, selecione o arquivo ".msi" (exemplo: node-vxx.x.x-x86.msi).
          • Baixe o arquivo e execute-o para instalar.
        1. Desinstale versões anteriores antes de instalar uma nova
          • Para evitar conflitos, desinstale a versão atual pelo Painel de Controle do Windows antes de instalar a nova versão.

        Instalar uma Versão Específica do Angular CLI

        1. Determine a versão desejada
          • Consulte as versões disponíveis do Angular CLI no site oficial ou na tabela de compatibilidade.
        1. Instale a versão específica

          Execute o seguinte comando no terminal:

          npm install -g @angular/cli@16.2.11

          Isso instala a versão 16.2.11 globalmente na sua máquina.

        1. Verifique a compatibilidade com o Node.js
          • Consulte a tabela de compatibilidade Angular-Node.
          • Por exemplo:
            • Angular CLI 16: compatível com Node.js 16 ou 18.
            • Angular CLI 17: compatível com Node.js 18 ou 20.

        Regras de Compatibilidade

        • Certifique-se de que a versão do Node.js instalada seja compatível com a versão do Angular CLI que você pretende usar.
        • Exemplo: Não é possível usar Node.js 14 para criar projetos na versão 16 ou 17 do Angular.

        Recomendações

        • Utilize a versão mais recente do Node.js (atualmente 20.x) e Angular CLI (atualmente 17.x) para evitar problemas de compatibilidade e manter-se atualizado com as boas práticas.
        • Se precisar de uma versão mais antiga do Angular ou Node.js, sempre verifique a tabela de compatibilidade disponível na documentação oficial.

        Conclusão

        Este vídeo demonstrou como instalar versões específicas do Node.js e Angular CLI, reforçando a importância da compatibilidade entre elas. Se ainda tiver dúvidas, veja a seção extra sobre NPM no curso para entender mais sobre versões Major, Minor, e Patch.

        • O Angular CLI é um pacote NPM que disponibiliza o comando ng para criar e gerenciar projetos Angular.
        • Comandos comuns:
          • Criar projetos Angular: ng new nome-do-projeto
          • Criar componentes, serviços e módulos: ng generate component, ng generate service, etc.
          • Executar projetos: ng serve

        2. Como Instalar o Angular CLI

        Passo 1: Instalar o CLI Globalmente

        Execute o comando abaixo para instalar a última versão do Angular CLI globalmente:

        npm install -g @angular/cli@latest

        Passo 2: Verificar a Instalação

        Após a instalação, verifique a versão instalada:

        ng version

        O comando exibirá:

        • Versão do Angular CLI.
        • Versão do Node.js e NPM.
        • Sistema operacional.
        • Pacotes Angular disponíveis (como @angular/core, etc.).

        3. Instalar Versões Específicas do Angular CLI

        Se você precisar de uma versão específica do Angular CLI, use uma tag no comando de instalação:

        • Instalar a versão 16:

        • Instalar uma versão específica, como 8.3.29:
          npm install -g @angular/cli@8.3.29

        • @latest: Última versão estável.
        • @v16-lts: Versão 16 LTS.
        • @v8-lts: Versão 8 LTS.

          • Angular CLI 16 → Cria projetos Angular na versão 16.

      • Aula 016 - Criação de um Projeto Angular

        Vamos detalhar o conteúdo ao máximo. Aqui está um passo a passo detalhado com os conceitos abordados no vídeo:

        Criação de um Projeto Angular com Angular CLI

        1. O que é o Angular CLI?

        O Angular CLI (Command Line Interface) é uma ferramenta que permite criar, gerenciar e executar projetos Angular de forma automatizada, facilitando o processo de desenvolvimento. Com ele, é possível:

        • Criar projetos, componentes, módulos, serviços e outras estruturas.
        • Inicializar projetos com configurações pré-definidas.
        • Executar servidores locais para desenvolvimento.
        • Construir e compilar projetos para produção.

        2. Criando o Projeto

        Comando de criação

        • O comando utilizado é:

          Esse comando cria toda a estrutura inicial do projeto.
          ng new NomeDoProjeto

        Etapas ao executar ng new:

        1. Escolha sobre rotas:
          • A ferramenta pergunta: “Deseja adicionar suporte a rotas no projeto?”.
          • Resposta típica: Yes (sim), caso pretenda usar rotas para navegação entre páginas.
        1. Seleção do tipo de estilo:
          • As opções incluem:
            • CSS: Estilos padrão.
            • SCSS: Extensão do CSS com recursos avançados (geralmente escolhido).
            • SASS ou LESS: Alternativas mais avançadas.
          • Escolha típica: SCSS.
        1. Instalação de dependências:
          • Após essas escolhas, o CLI cria a estrutura inicial e instala automaticamente todas as dependências do projeto, baixando pacotes definidos no package.json.

        Estrutura do Projeto Angular

        Após a execução do comando, uma pasta com o nome do projeto é criada contendo os seguintes arquivos e diretórios principais:

        1. Estrutura geral

        • package.json:
          • Gerencia todas as dependências do projeto, como bibliotecas do Angular, RxJS, TypeScript, entre outras.
          • Exemplo de dependências:
            • @angular/core: Pacote central do Angular.
            • rxjs: Biblioteca reativa usada pelo Angular.
            • zone.js: Utilizado para rastrear e gerenciar mudanças de estado na aplicação.
        • angular.json:
          • Configuração principal do projeto:
            • Define os estilos globais (CSS/SCSS).
            • Configura os assets (imagens, ícones, etc.).
            • Especifica arquivos de entrada como main.ts e index.html.
        • src (source):
          • Diretório principal onde fica o código do projeto.

        2. Pasta src

        Arquivos principais

        1. main.ts:
          • Ponto de entrada da aplicação.
          • Responsável por carregar o módulo principal (AppModule).
          • Exemplo de código:

            Esse código inicializa o módulo principal (Bootstrap).
            platformBrowserDynamic().bootstrapModule(AppModule);
        1. index.html:
          • Arquivo HTML principal.
          • Contém o seletor do componente inicial:
            <app-root></app-root>

        Pasta app

        • app.component.ts:
          • Código do componente inicial da aplicação.
          • Define:
            • Seletor: Nome utilizado no HTML para representar o componente (app-root).
            • Template: HTML associado ao componente.
            • Estilo: Arquivo CSS/SCSS específico do componente.
        • app.module.ts:
          • Declaração de todos os componentes do projeto.
          • Define qual componente será inicializado (via propriedade bootstrap).

        Inicializando o Projeto

        Para rodar a aplicação localmente:

        1. Execute o comando:
          ng serve
        1. Isso faz com que:
          • Um servidor local seja iniciado.
          • O projeto seja acessado em http://localhost:4200.

        Processo de Bootstrap no Angular

        O termo Bootstrap refere-se à inicialização do projeto. Entenda o fluxo:

        1. O Angular inicia no arquivo main.ts.
        1. O arquivo main.ts carrega o módulo inicial (AppModule).
        1. O AppModule tem uma propriedade chamada bootstrap, que define o componente inicial (geralmente AppComponent).
        1. O AppComponent é renderizado no seletor especificado no index.html (<app-root>).

        Testes Práticos

        1. Alteração de seletores

        • Exemplo:
          • Alterar o seletor do AppComponent de app-root para app-teste.
          • Atualizar o index.html para:
            <app-teste></app-teste>
          • Caso não seja atualizado, o Angular gera um erro informando que o seletor não foi encontrado.

        2. Criando um novo componente

        • Para criar um componente:
          ng generate component NomeDoComponente

          Isso gera:

          • Uma pasta com os arquivos do componente (.html, .scss, .ts, .spec.ts).
          • Atualiza automaticamente o módulo principal (AppModule), adicionando o novo componente.
        • O novo componente pode ser usado:
          • No AppComponent template:
            <app-novo-componente></app-novo-componente>

        3. Carregando múltiplos componentes

        • Teste:
          • Adicionar um segundo componente ao bootstrap no AppModule.
          • Resultado:
            • O Angular espera encontrar ambos os seletores no index.html.

        Considerações Finais

        • A estrutura do Angular pode parecer complexa inicialmente devido à divisão de responsabilidades entre componentes, módulos e serviços.
        • Com o tempo, esses conceitos tornam-se naturais, permitindo a construção de aplicações altamente escaláveis.
        • A prática de criação de componentes e alterações nos módulos ajuda a consolidar os conceitos apresentados.

        Espero que este resumo mais detalhado esclareça todos os pontos abordados no vídeo! Se desejar mais explicações ou exemplos práticos, é só pedir.

            let teste = 'Exemplo'; // Copie esta linha com Ctrl + C e cole com Ctrl + V.
        1. Salvar arquivo:
          • Ctrl + S: Salva manualmente o arquivo.
          • Auto Save:
            • Por padrão, o VSCode pode salvar automaticamente após um intervalo.
            • Para desativar:
              1. Acesse Settings (Ctrl + , ou engrenagem no canto inferior esquerdo).
              1. Pesquise por Auto Save.
              1. Altere para off para salvar manualmente.
        1. Desfazer e refazer alterações:
          • Ctrl + Z: Desfaz a última alteração.
          • Ctrl + Y: Refaz a alteração desfeita.

        Atalhos de Comentários

        1. Comentar/Descomentar linhas:
          • Ctrl + ; (ponto e vírgula):
            • Adiciona ou remove comentários em uma linha ou seleção de linhas.
          • Funciona com múltiplas linhas:
            // Exemplo antes de comentar:
console.log('Linha 1');
console.log('Linha 2');

// Após Ctrl + ;:
// console.log('Linha 1');
// console.log('Linha 2');

        1. Abrir/Fechar terminal:
          • Ctrl + ` (apóstrofo):
            • Abre ou fecha o terminal integrado.

          • Ctrl + B:
            • Abre ou fecha o painel lateral do Explorer, permitindo navegar entre os arquivos.
        1. Pesquisar texto em um arquivo:
          • Ctrl + F:
            • Permite encontrar texto específico dentro de um arquivo.
        1. Pesquisar arquivos por nome:
          • Ctrl + P:
            • Permite localizar rapidamente um arquivo pelo nome.
            • Exemplo:
              • Digite app para encontrar arquivos como app.component.ts.

          • Alt + Shift + ↓ (seta para baixo):
            • Duplica a linha ou seleção para baixo.
          • Alt + Shift + ↑ (seta para cima):
            • Duplica a linha ou seleção para cima.
        1. Mover linhas:
          • Alt + ↓ (seta para baixo):
            • Move a linha atual para baixo.
          • Alt + ↑ (seta para cima):
            • Move a linha atual para cima.
        1. Selecionar textos iguais:

}
